# MCP Docs Server — простыми словами

## Что это такое?

MCP Docs Server — это локальный помощник для программиста, который умеет **читать документацию за тебя** и **давать модели только нужные куски**.

Он работает вместе с Continue.dev и локальными LLM.

---

## Зачем он нужен?

Когда документации много:

- она не влезает в контекст модели
- поиск по ней неудобен
- модель отвечает «наугад»

Этот сервер:

- скачивает документацию один раз
- хранит её локально
- быстро находит нужные места
- автоматически передаёт их агенту

---

## Как это выглядит в работе?

Ты задаёшь вопрос в Continue:

> «Как работает fs.watch в Node.js?»

Агент:

1. Сам вызывает MCP Docs Server
2. Делает поиск по документации
3. Собирает релевантные куски
4. Отвечает с точными цитатами

Ты ничего вручную не делаешь.

---

## Нужно ли разбираться в коде?

Нет.

Достаточно:

- один раз указать ссылки на документацию
- запустить сервер

Дальше всё происходит автоматически.

---

## Где хранятся данные?

Всё локально на твоём компьютере:

- документы
- индекс
- ошибки

Ничего не отправляется в интернет, кроме загрузки документации.

---

## Чем это НЕ является?

- Это не чат‑бот
- Это не облачный сервис
- Это не база знаний

Это **инструмент для агентов**.

---

## Когда это особенно полезно?

- Большие фреймворки
- Плохо структурированные доки
- Локальные LLM
- Работа без интернета

---

## Итог

MCP Docs Server — это:

✔ локально
✔ автоматически
✔ точно
✔ без ручной возни

Он делает так, чтобы агент **думал с документацией, а не вместо неё**.
